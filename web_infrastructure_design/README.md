# Web Infrastructure Design

## Description

This project focuses on designing and understanding web infrastructures of increasing complexity. The objective is to learn how web applications are hosted, how different components interact, how to identify single points of failure (SPOF), and how to improve scalability, security, availability, and monitoring.

The project covers:

* Simple web stack architecture
* Distributed web infrastructure
* Secured and monitored infrastructure
* Scaled-up infrastructure

All diagrams were designed to illustrate how a user request travels through the system and how each component contributes to the overall architecture.

---

## Learning Objectives

At the end of this project, I am able to explain:

* What a server is
* The role of a web server
* The role of an application server
* The role of a database
* How DNS works
* How load balancing works
* The difference between Active-Active and Active-Passive setups
* How Primary-Replica database replication works
* What HTTPS is and why it is important
* What firewalls are used for
* What monitoring is used for
* How to identify Single Points of Failure (SPOF)
* How to improve availability and scalability
